## Trap: the rules-vs-bootstrap circular dependency

The #1 gotcha in this project:

- RT-SC's `firestore.rules` has `isAdmin()` checking for a doc at
  `/professeurs/{uid}` with `role: 'admin'`
- A fresh project has no such doc → rules reject all writes including
  the bootstrap's own writes
- If you deploy full mode (`./deploy-school.sh <config>`) on a fresh
  project, rules get pushed BEFORE bootstrap → bootstrap fails →
  you're locked out

**Solution**: always deploy hosting first (`--hosting-only`), then
bootstrap via vendor-app (while rules are default permissive
test-mode), then deploy rules (`--rules-only`) to lock down.

This is documented in DEPLOY-PLAYBOOK.md. Don't forget it. Future
deploy-school.sh should auto-detect bootstrapped state and pick the
right mode; doing so is a deferred improvement.
